Mathematical Core

Cryptographic Mathematics

A deep dive into the underlying theoretical frameworks of the Hybrid Quantum-Safe Secure Communication Platform.

Classical: X25519 (Elliptic Curve)

X25519 uses Elliptic Curve Diffie-Hellman (ECDH) over Curve25519. It relies on the computational difficulty of the Elliptic Curve Discrete Logarithm Problem (ECDLP).

The curve is defined over a prime field \(\mathbb{F}_p\) by the Montgomery equation:

\[ y^2 = x^3 + 486662x^2 + x \pmod p \]

Where the prime \(p\) is defined as:

\[ p = 2^{255} - 19 \]

Key Exchange Mechanism

Alice and Bob agree on a base point \(G\). They choose private keys \(a\) and \(b\), generating public keys \(A = aG\) and \(B = bG\).

The shared secret \(K\) is computed as:

\[ K = aB = a(bG) = b(aG) = bA \]

Vulnerability

While extremely fast (~1ms), ECDLP can be solved in polynomial time by a sufficiently large quantum computer using Shor's Algorithm, making it vulnerable to "Store-Now-Decrypt-Later" attacks.

Post-Quantum: ML-KEM-768 (Lattice-Based)

ML-KEM-768 (formerly CRYSTALS-Kyber) is standardized by NIST (FIPS 203). It is based on the hard problem of Module Learning with Errors (M-LWE) over polynomial rings.

Let \(R_q = \mathbb{Z}_q[X]/(X^n + 1)\) be a polynomial ring where \(n = 256\) and the modulus \(q = 3329\). For ML-KEM-768, we work with vectors/matrices of dimension \(k = 3\) over \(R_q\).

The core mathematical relationship (the public key generation) is:

\[ \mathbf{t} = \mathbf{A} \cdot \mathbf{s} + \mathbf{e} \pmod q \]
  • A: A randomly generated \(k \times k\) matrix in \(R_q\).
  • s: The secret key vector of size \(k\), sampled from a centered binomial distribution (small coefficients).
  • e: An error vector of size \(k\), also sampled from a binomial distribution.
  • t: The resulting public key vector.

Security Premise

The security relies on the premise that given \(\mathbf{A}\) and \(\mathbf{t}\), it is computationally intractable (even for quantum computers) to recover the secret \(\mathbf{s}\) due to the deliberate addition of the small error \(\mathbf{e}\).

Encapsulation

During encapsulation, the sender generates a random message \(m\), an ephemeral secret \(\mathbf{r}\), and errors \(\mathbf{e_1}\), \(e_2\). The ciphertext consists of \((\mathbf{u}, v)\):

\[ \mathbf{u} = \mathbf{A}^T\mathbf{r} + \mathbf{e_1} \] \[ v = \mathbf{t}^T\mathbf{r} + e_2 + \lfloor q/2 \rceil \cdot m \]

Hybrid Combining: HKDF-SHA256

To ensure resilience, the secrets from both X25519 (\(S_{ECC}\)) and ML-KEM-768 (\(S_{PQC}\)) are concatenated and fed into an HMAC-based Key Derivation Function (HKDF).

1. Extract Phase: Generates a fixed-length pseudorandom key (PRK).

\[ PRK = \text{HMAC-Hash}(\text{salt}, S_{ECC} \Vert S_{PQC}) \]

2. Expand Phase: Derives the final 256-bit symmetric session key.

\[ Key_{session} = \text{HMAC-Hash}(PRK, \text{info} \Vert \text{0x01}) \]

Because HKDF is a one-way function, an attacker must break both the classical ECDLP and the post-quantum M-LWE problem to recover the session key. If ML-KEM-768 has an unforeseen mathematical vulnerability, X25519 still protects the channel against classical attacks.

Symmetric Encryption: AES-256-GCM

Once the hybrid handshake generates the session key, real-time WebSocket messages are encrypted client-side using Advanced Encryption Standard (AES) in Galois/Counter Mode (GCM).

GCM provides both confidentiality and data origin authentication (AEAD).

Encryption (Counter Mode):

\[ C_i = P_i \oplus E_K(CB_i) \]

Where \(P_i\) is plaintext block \(i\), \(E_K\) is AES encryption with the session key, and \(CB_i\) is the counter block generated from the 12-byte Initialization Vector (IV).

Authentication (Galois Hash):

GCM computes an authentication tag \(T\) by evaluating a polynomial over the Galois field \(GF(2^{128})\). The ciphertext blocks and any Additional Authenticated Data (AAD) form the coefficients of this polynomial, which is evaluated at a hash subkey \(H = E_K(0^{128})\).

\[ T = (\text{GHASH}_H(A, C) \oplus E_K(CB_0)) \]

This 16-byte tag ensures that any tampering with the ciphertext in transit across the WebSocket relay is immediately detected, causing the decryption to fail.
